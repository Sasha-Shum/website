Teleporter

Language: Swift Platform: Xcode 7.1 Year: 2016 Team Project (2 people)

Description:

Teleporter is a defense based arcade game. The main goal is to protect your space ship from asteroids using a teleporter that you can control.

Mechanics:

The player can’t move the space ship, but they have an ability to create a teleporter by tapping on the screen. The teleporter can catch an asteroid and fire it in order to destroy other asteroids. The teleporter can be rotated 90 degrees and can contain only one asteroid. If the player fails to protect their ship, the game is over. Once all the asteroids are destroyed, the new wave of countdown appears and notifies the player about upcoming new wave.

The Creature: Subject To Change

Language: C# Platform: Unity Year: 2016 Team Project (6 people. 3 Designers 3 Programmers)

Description:

“The Creature” is a 3D adventure game based on the story of Frankenstein, where you are a creature created in the lab and whose father was killed by villagers. Now you seek revenge and nothing stops you.

Mechanics:

The game allows the player to change the creature's body parts. Each body part has a fixed amount of health and can be destroyed. By killing enemies, the player can pick up the enemies' body parts and attach them instead of theirs. Switching body parts is done through a mouse click and the inventory system. The creature also has a grappling hook allowing to grapple to specific points. Also, the grappling hook can be used to grab the enemy and pull them towards the creature. The game has 8 levels and each level requires different actions from the player.

My Work:

During this project I was responsible for the visual design of the menus, visual effects, inventory system and grappling hook functionality.

GravityBall

Language: Swift Platform: Xcode 7.1 Year: 2016 Solo Project

Description:

GraviBall is a physics-based arcade game where the user controls a gravitational ball, tries to collect all the stars (collectibles) on the level as well as hidden elements and tries to find the way out of the level.

Mechanics:

The gameplay mechanics reminds of the one in “Flappy Bird”, where the user have to tap on the screen to move the ball. The game has six levels. Each level is unique and adds more and more elements as you progress. The light in the game plays a crucial part, because it helps to see not only what is around you but also where the hidden elements are. If you get close enough to a hidden element, it will appear on the screen. If you move away, it will disappear again. Also, as you get closer to collectibles, they will be attracted by you and will move toward you. Some of the traps in the game will also move toward you as you get closer to them. Some of them will just push you back and some traps will re-size you and make you smaller.

Challenge:

The main challenge of this project was to balance the game as well as levels and to create fun and entertaining gameplay. The project was created in a short amount of time. Since it was a solo project, I had to spend a lot of time on game visuals. I also had to learn how to work with swift physics and tried to use variety of it in order to create good mechanics for the game.

Zombie World

Language: C# Platform: Unity Year: 2015 Solo Project

Description:

Zombie world is a first person survival playground. The main goal is to avoid zombies and try to save as many soldiers as you can by finding a switch allowing to open the city's gates.

Mechanics:

This project utilizes different steering behaviors for AI such as: pursue, evade, wander, arrival to the point and flocking. Zombies in the city try to chase you or soldiers as soon as they see you. The visual field of a zombie is 120 degrees, so they detect the player or soldiers if they are in the radius of the zombies’ vision. Otherwise, they just wander around. Soldiers also wander around until they see zombies. As soon as a zombie gets closer to the soldiers, the soldiers try to evade. If the player activates the pressure plate, which opens the gates, the soldiers will try to evade as well as run toward the safe zone behind the gates.

Challenge:

The main challenge in this project was to resolve collision detection. Due to a very large number of obstacles on the map, AI was getting stuck all the time. Also, it was quite a challenge to design a level and make it convey “the end of the world” feeling to the player.

Save The Bouncers

Language: Java Script Platform: Phaser Year: 2016 Solo Project

Description:

Save The Bouncers is an arcade game similar to “Mario”. Your goal is to collect all the bouncers on the map. The player has to avoid enemies and traps as they instantly kill the player.

Mechanics:

The game controls are similar to the ones in “Mario”. You are in charge of the main character collecting coins and bouncers. Jumping on top of the enemies kills them and bounces the main character up. Also, the player can perform a double jump using coins, enemies and bouncers for the second jump. The traps have different speeds and timing and therefore create a challenge for the player.

Challenge:

I had to educate myself how to use the Phaser framework as well as the software that help to create a map and generates a JSON file with location of each tile on the map. Also, I spent a lot of hours working on the level design and balance.

Life Commander

Language: C# Platform: WPF & Visual Studio Year: 2013 Group Project (3 people)

Description:

Life Commander is a life simulation game with some micromanagement. The player's main goal is to manage the character's life and live as long as they can. The character gets older every year and the player has to micromanage their life: send them to school, to work, cure the character if they get sick, move to another country, buy property etc. But for every action the player does, they should have enough money to pay for it.

Mechanics:

The player has one main button and a lot of buttons for micromanagement. One time clicking on the main button allows the player to progress a year, while clicking on the buttons for micromanagement allows the character to attend school, to travel, to buy property, etc.

My Work:

During this team project, I was working on game visual aspects as well as the character creation menu, map, travel system and custom Windows Forms. Also, for the travel system and character creation, I had to work with a database.

Challenge:

We wanted to create something new and unique for this project. Therefore, I decided to create custom Windows Forms using WPF. The main challenge was to create an interactive map. I wanted the player's clicking on any part of the map to find the country clicked on in the database and to move the player to that country. Also, in order to show the player where they point at, I had to make each region to have a selected effect (change the color). The map was created by using SVG image and converted to the XAML,and then I had to associate each path with the proper country and make it interactive.
